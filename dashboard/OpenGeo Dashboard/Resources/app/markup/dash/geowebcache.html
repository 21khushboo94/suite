GeoWebCache

GeoWebCache intelligently stores and serves map tiles using standard web protocols to make map services more efficient and scalable.

Accelerate the performance of your maps with GeoWebCache. Traditional map servers generate a new image on every request. GeoWebCache sits between GeoServer (or any compliant WMS) and a tiling client (like OpenLayers or GeoExt) and saves previously requested tiles. Once tiles are stored in GeoWebCache, maps are returned to the client up to 100 times as fast as traditional map servers.

GeoWebCache includes an administration console to manage a layer's tiles. From this console, it is possible to 'seed' a layer, which will make a series of requests to GeoServer to generate tiles, ensuring that the maps are pre-rendered for a better user experience.

GeoWebCache supports tiling standards like Tiled Map Service (TMS) and Web Mapping Service-Cached (WMS-C). Support for the Web Map Tile Server (WMTS) will be available in a future release. GeoWebCache is also compatible with other tile schemes. Google Earth, Google Maps, Bing (Virtual Earth), and Mobile GMaps can all access GeoWebCache tiles directly.

Further information is available in the GeoWebCache Documentation.
