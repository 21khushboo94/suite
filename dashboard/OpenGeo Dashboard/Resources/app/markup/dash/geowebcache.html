GeoWebCache

GeoWebCache intelligently stores and serves map tiles using standard web protocols to make map services scale.

Accelerate the performance of your maps with GeoWebCache. Traditional map servers generate a new image on every request. GeoWebCache sits between GeoServer (or any compliant WMS) and a tiling client (like OpenLayers or GeoExt), and saves already requested images. Once tiles are stored in GeoWebCache, maps are returned to the client 10 to 100 times as fast as traditional map servers.

GeoWebCache includes an administration console to 'seed' a layer. This makes a series of requests to GeoServer, saving rendered images thereby ensuring that the first user experience is fast from the beginning.

GeoWebCache supports tiling standards like TIGER Map Service (TMS) and Web Mapping Service-Cached (WMS-C). Support for the Web Map Tile Server (WMTS) will be available in a future release. GeoWebCache is also compatible with other tile schemes. Google Earth, Google Maps, Bing (Virtual Earth), and Mobile GMaps can all access GeoWebCache directly.

Further information is available in the GeoWebCache Documentation.
