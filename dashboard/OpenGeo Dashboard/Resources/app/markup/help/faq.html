Frequently Asked Questions

Here you can find answers to the most common questions asked about the OpenGeo Suite. More component-specific questions can be answered by looking at the individual component documentation.

If something goes wrong, how do I find out the error?

If you have unexpected behavior, the most helpful place to look is the Logs. Although the logs can be verbose, errors are displayed in line amongst other status outputs. You may have to scroll back to find the error you're looking for.

If I have trouble, who can I contact?

Please email inquiry@opengeo.org for any questions. If you are are one of our valued customers, we provide technical support for all aspects of this software. For sales information, please contact inquiry@opengeo.org as well.

What are the technical specs of the OpenGeo Suite?

The OpenGeo Suite is a collection of Java servlet containers, running inside a Jetty web server.

Is this the full OpenGeo Suite stack?

The full OpenGeo Suite includes support for PostGIS, a spatial database. This software natively supports and connects to existing PostGIS databases. Future versions of this software will include PostGIS natively.

How do I learn more about the Suite?

If you are new to the OpenGeo Suite, please see the Getting Started section. We have also included the full-up-to-date documentation for all components of the Suite. If your questions aren't answered in these places, feel free to contact us at inquiry@opengeo.org .

What do I do if I want to implement the OpenGeo Suite in my business?

The OpenGeo Suite is designed to be used in a production environment, however, the evaluation version is not supported. Please contact inquiry@opengeo.org for sales opportunities.

Where do I go if I have a feature request?

We are always looking to improve our software, and our improvements are always driven by customer needs and desires. Please contact inquiry@opengeo.org with your requests. Existing customers' requests are given highest priority.

Where can I learn more about OpenGeo/GeoServer/GeoExt/etc?

Please visit our website, at opengeo.org . You can also visit each community project's homepage: PostGIS, GeoServer, GeoWebCache, OpenLayers, and GeoExt. We also encourage those interested in a particular project to join that project's mailing list, to interact with the wider community.

Isn't PostGIS a part of the OpenGeo Suite?

Yes. This software natively supports and connects to existing PostGIS databases. Future versions of this software will include PostGIS natively.

How do I start the OpenGeo Suite?

There are a few ways to start the OpenGeo Suite. The simplest way is through the Dashboard, by clicking the Start button on the bottom right on the page. You can also manually start and stop the suite by running the "opengeo-suite" script in the root of your application directory.

Why can't I start the OpenGeo Suite?

The most common reason why the OpenGeo Suite is unable to start is because of a port conflict. By default, the OpenGeo Suite runs on port 8080 of the host machine, and if another service using that port, the Suite will not run. To change the OpenGeo Suite port, go to the Preferences tab, change the (NAME OF FIELD) click Save, then restart the OpenGeo Suite. If you are still unable to start the service, you may want to check the Logs for other errors. If you are still unable to find you answer, please contact us at inquiry@opengeo.org with a description of your issue.

How do I change the port that the OpenGeo Suite runs on?

GREAT QUESTION

Where is my GeoServer data directory?

Your GeoServer data directory is located inside the root of your installation, in a directory called "data_dir".

Why did I get a Java stack trace with GeoServer on my web browser?

Java stack traces can happen for many reasons, ranging from incorrect requests to software bugs. The first lines of the stack trace normally have the most helpful information. If you are unable to discern to particular error you're receiving, please copying into an email and send it to inquiry@opengeo.org .

How do I add lots of data to the OpenGeo Suite at once?

The OpenGeo Suite includes a Layer Importer as an extension to GeoServer that allows for batch loading of PostGIS, Oracle, and ArcSDE tables, and also shapefiles.

Why is my OpenLayers map blank (or pink)?

You may wish to look at the Error Console for your browser, or even better, use a JavaScript debugger. If using Mozilla Firefox, a popular debugger is Firebug.

Does Styler include all of the functionality contained in the SLD spec?

Style includes most common uses contained in the SLD specification. Certain extras, such as vendor options, are not preserved. If unsure about editing an existing SLD file with Styler, please make a backup first.

Why are my vendor options are not rendered when I view an existing SLD in Styler?

Styler does not recognize vendor options, which are not part of the SLD specification.
