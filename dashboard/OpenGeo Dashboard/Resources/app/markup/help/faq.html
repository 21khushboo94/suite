Frequently Asked Questions

Here you can find answers to the most common questions asked about the OpenGeo Suite. More component-specific questions can be answered by looking at the individual component documentation.

If something goes wrong, how do I find out the error?

If you have unexpected behavior, the most helpful place to look is the Logs. Although the logs can be verbose, errors are displayed in line among other status outputs. You may have to scroll up to find the error you're looking for.

If I have trouble, who can I contact?

Please email help@opengeo.org or submit to our support page for any questions. If you are are a subscriber to The OpenGeo Suite, Enterprise Edition, we provide technical support for all aspects of this software. For sales information, please contact inquiry@opengeo.org as well.

Can I get support if I'm using the 30-day Enterprise Edition trial?

Yes, the trial comes with support, up to three incidents (with more available on request). Just be sure to send your support request from the email you registered with.

Can I get support if I'm using the Community Edition?

No, the Community Edition does not include professional support. If the question is limited to one of the individual open source projects please email the project's user list for assistance.

What are the technical specs of the OpenGeo Suite?

The OpenGeo Suite is a collection of Java servlet containers, running inside a Jetty web server. Client-side code is written in JavaScript.

Is this the full OpenGeo Suite stack?

The full OpenGeo Suite includes support for PostGIS, a spatial database. This software natively supports and connects to existing PostGIS databases. Future versions of this software will include PostGIS.

How do I learn more about the Suite?

If you are new to the OpenGeo Suite, please see the Quickstart or the Getting Started documentation. We have also included the full up-to-date documentation for all components of the Suite. If your questions aren't answered in these places, Enterprise Edition customers and trial users are free to contact us at email help@opengeo.org or submit to our support page.

What do I do if I want to implement the OpenGeo Suite in my agency or business?

The OpenGeo Suite is designed to be used in a production environment, and the Enterprise Edition ensures that you have all the support you need to successfully utilize and maintain a deployment. Please contact inquiry@opengeo.org for sales opportunities.

Where do I go if I have a feature request?

We are always looking to improve our software, and our improvements are always driven by customer needs and desires. Please contact inquiry@opengeo.org with your requests. Existing customer requests are given highest priority.

Where can I learn more about OpenGeo/GeoServer/GeoExt/etc?

Please visit our website at opengeo.org. You can also visit each community project's homepage: PostGIS, GeoServer, GeoWebCache, OpenLayers, and GeoExt. We also encourage those interested in a particular project to join that project's mailing list to interact with the wider community.

How do I file a bug report?

If you are on an Enterprise Edition (either trial or full contract) please email help@opengeo.org or submit to our support page. Community Edition users should file reports with the components' open source projects, according to their conventions.

Is PostGIS a part of the OpenGeo Suite?

Yes. This software natively supports and connects to existing PostGIS databases. Future versions of this software will include PostGIS.

Does the OpenGeo Suite support connections with ArcSDE databases?

Yes, however there are certain required files that we are not allowed to redistribute. To enable ArcSDE support, copy jsde*.jar and jpe*.jar from your ArcSDE installation into the installation directory under "webapps/geoserver/WEB-INF/lib" and restart. You should then see ArcSDE as an option in GeoServer.

Does the OpenGeo Suite support connections with Oracle Spatial databases?

Yes, however there are certain required files that we are not allowed to redistribute. To enable Oracle Spatial support, copy ojdbc*.jar from your Oracle installation into the installation directory under "webapps/geoserver/WEB-INF/lib" and restart. You should then see Oracle as an option in GeoServer.

How do I start the OpenGeo Suite?

There are a few ways to start the OpenGeo Suite. The simplest way is through the Dashboard, by clicking the Start button on the bottom right on the page. You can also manually start and stop the suite by running the "opengeo-suite" script in the root of your application directory.

Why can't I start the OpenGeo Suite?

The most common reason why the OpenGeo Suite is unable to start is because of a port conflict. By default, the OpenGeo Suite runs on port 8080 of the host machine, and if another service using that port, the Suite will not run. To change the OpenGeo Suite port, go to the Preferences page, change the Primary Port, click Save, then restart the OpenGeo Suite. If you are still unable to start the service, you may want to check the Logs for other errors. If you are still unable to find your answer, please contact us at our support page or email help@opengeo.org with a description of your issue.

How do I change the port that the OpenGeo Suite runs on?

By default, the OpenGeo Suite runs on port 8080 of the host machine. To change the OpenGeo Suite port, go to the Preferences page, change the Primary Port, click Save, then restart the OpenGeo Suite.

How do I make sense of these logs?

Many Java errors are very long and span multiple lines (all usually being with the word "at ..."). Usually the first line of the error is where the human-readable content is. You may have to scroll up to find the error you're looking for.

Where is my GeoServer data directory?

Your GeoServer data directory is located inside your user's home directory at ".opengeo/data_dir".

Why did I get a Java stack trace with GeoServer on my web browser?

Java stack traces can happen for many reasons, ranging from incorrect requests to software bugs. The first lines of the stack trace normally have the most helpful information. If you are unable to discern to particular error you're receiving, please copy it into an email and send it to help@opengeo.org or submit to our support page.

How do I add lots of data to the OpenGeo Suite at once?

The OpenGeo Suite includes a Layer Importer as an extension to GeoServer that allows for batch loading of database tables and shapefiles.

Why is my OpenLayers map blank (or pink)?

You may wish to look at the Error Console for your browser, or even better, use a JavaScript debugger. If using Mozilla Firefox, a popular debugger is Firebug.

Does Styler include all of the functionality contained in the SLD spec?

Styler includes most common uses contained in the SLD specification. Certain extras, such as vendor options, are not preserved. If unsure about editing an existing SLD file with Styler, please make a backup first.

Why are my vendor options are not rendered when I view an existing SLD in Styler?

Styler does not recognize vendor options, which are not part of the SLD specification.

How do I increment the bookmark number in GeoExplorer?

The bookmark number increments automatically when you create a new map.
