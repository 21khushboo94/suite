Styling¶

Basic Styling¶

  1. With Basic styling, you can change SLD parameters including, but not limited to Name, Stroke, Fill and Symbol. Options vary according to geometry type. The following figure displays the three types of Basic styling forms, for point, polygon, and line respectively.

    Basic Panel for a Zone Polgyon Style

  2. Parameters options can be changed as follows:

    • Name: type in the Name text field

    • Symbol: for Point styling, select between a circle, square, triangle, star, cross, X, and custom. . from the drop down menu.

      Choosing a Point Symbol

    • Size: for Point styling, set the absolute size of the graphic in pixels

    • Rotation: Define the clockwise rotation of the point, in decimal degrees.

    • Color: choose a stroke color by clicking the Color text field.

      Choosing a Stroke Color

    • Opacity: set color transparency using the slider bar, with zero fully transparent, and 100% fully opaque.

    • Style: for Stroke styling, select between a solid, dashed, or dotted line from the drop down menu

    • Width: for Stroke styling, type in a stroke width in pixels.

Label Styling¶

The Labels tab defines how to display text labels. Options set here correlate to a block of code in the SLD called the <TextSymbolizer>.

  1. To enable label editing, select the Labels tab and check :guilabel: check Label Features.

    Enabling Style to edit label features

  2. Label Values: defines what label to give your style. Options refer to attributes specified in your data and can be significant in number. For example, the RailRoad Border layer has over 50 possible values. Select the Label values that makes the most sense for your style. For RailRoad Border we select the value namelow.

    Selecting a label value for RailRoad Borders

  3. Type styling including Font, Size, Bold, and Italic determine the–not surprisingly–font, size and weight of a label.

  4. Fill options specify the Color and Opacity that we will fill the font with.

  5. Enable Halo by check-marking the Halo box.

    Enabling Halo

  6. A halo fills an extended area outside the text label to make the label easier to read over a background. Essentially it is buffer outline around the text. Set the Size (in pixels), the Color, and Opacity of the Halo in the same manner as setting Basic styling parameters.

Advanced Styling¶

The Advanced tab allows for two possible filtering. Limiting by scale means styles can display differently depending on the maps zoom level. Limiting by condition, usually based on a layer attribute value, restricts what parts of the layer the style applies to.

Scale Filters¶

We recommend using the Zoom Slider on the upper left corner when setting up scale-dependent styling. At any time the current scale can be viewed by hovering over the zoom slider.

Viewing zoom level

The Min scale limit and the Max scale limit define the range of map-rendering scales for which the rule should be applied. The minimum scale refers to the smallest scale ratio, while the maximum scale the largest. In other words, the closer you are visually to a map feature the smaller the scale limit. For example, the image on the left displays the Medford zone and buildings layer at a scale of 1:4625, while the image of the right displays the same zones at a scale of 1:68247.

Different scale of Medford layers. On the left a 1:4625, on the right 1:68247

  1. To enable filtering by scale, check the Limit by scale checkbox.

    Enabling scale filtering

  2. To enter your scale range, enable the Min and Max checkboxes and enter your scale amounts in the text fields. To filter the range above, we can use the rounded values of 4000 and 70000.

    Creating a scale filter for the range 4K-70K

Condition Filters¶

  1. To enable filtering by condition, check the Limit by condition checkbox.

    Enabling condition filtering

  2. The first drop down menu, :guilabel::Match, specifies which kind of semantic operator you’re using. Options include, any, or, and and. These relate respectively, to <Or>, <And>, and <ElseIf> filters.

  3. Choose the layer attribute to restricts styling rules, with the class drop down menu.

    Selecting a layer attribute for filtering

  4. Select your comparison operator in the following drop down menu. Options include: - (=) PropertyIsEqualTo - (<>) PropertyIsNotEqualTo - (<) PropertyIsLessThan - (>) PropertyIsGreaterThan - (<=) PropertyIsLessThanOrEqualTo - (>=) PropertyIsGreaterThanOrEqualTo - (like) PropertyIsLike

  5. Type the value for filtering in the final text box. The condition in the following RailRoad example is sld_type = rail_roads.

    Defining a condition filter

  6. For more advanced filtering, select the add condition filter to append multiple conditions.
